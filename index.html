Welcome to C Web Server Benchmarking Simulator

For First Leg, we will be analyzing a TCP Server with 3 flavors.

  • Single Threaded Server
  • Multithreaded Server
  • Event Loop Based Server

There are 4 routes that I've built, each performing a different load on the server:

  • IO Intensive
  • Compute Intensive
  • Sends a simple HTML Page

Different web servers have different strategies on how they handle incoming traffic. These strategies can include single-threaded, multithreaded, or event-loop based approaches.

All the web servers are very rudimentary, written in C, to avoid benchmarking a server where optimizations by the framework or the programming language can pollute the results.
